# HH Analytics: Анализ рынка вакансий аналитиков

Pet-проект по сбору, обогащению и визуализации данных о вакансиях аналитических ролей с HeadHunter.

---

## Цель проекта

Построить аналитический пайплайн для исследования рынка вакансий аналитиков в РФ:

- Динамика зарплат (гросс и нетто)
- Распределение по регионам
- Требуемые навыки и профессиональные роли
- Анализ работодателей (отрасли, аккредитация)

---

|----------|------------|

## Схема данных (Supabase)

### Таблица exchange_rates

Курсы валют к рублю (1 единица валюты = rate_to_rub рублей).

| Поле | Тип | Описание |
|------|-----|----------|
| id | SERIAL | Первичный ключ |
| currency_code | TEXT | Код валюты (USD, EUR, BYN, KZT, UZS, AMD, KGS) |
| currency_name | TEXT | Полное название |
| rate_to_rub | DECIMAL(15,6) | Курс за 1 единицу |
| nominal | INTEGER | Номинал (всегда 1) |
| rate_date | DATE | Дата курса |
| source | TEXT | Источник ('cbr') |
| created_at | TIMESTAMPTZ | Дата создания записи |
| updated_at | TIMESTAMPTZ | Дата последнего обновления |

**Уникальность:** сочетание (currency_code, rate_date)

---

### Таблица vacancies (49 полей)

**Базовые**
- id (TEXT, PRIMARY KEY) — уникальный ID вакансии из HH
- name (TEXT) — название вакансии
- published_at (TIMESTAMPTZ) — дата публикации (ключевая для аналитики)
- created_at (TIMESTAMPTZ) — дата создания/последнего изменения
- initial_created_at (TIMESTAMPTZ) — истинная дата создания (если есть)
- alternate_url (TEXT) — ссылка на вакансию

**Зарплата (исходная)**
- salary_from (NUMERIC(12,2)) — нижняя граница в исходной валюте
- salary_to (NUMERIC(12,2)) — верхняя граница в исходной валюте
- salary_currency (TEXT) — валюта (USD, EUR, BYR, KZT, UZS, AMD, KGS)
- salary_gross (BOOLEAN) — true = до вычета налогов

**География**
- area_id (TEXT) — ID региона/города из справочника HH
- area_name (TEXT) — название региона/города
- lat (DECIMAL(10,6)) — широта (из адреса или справочника городов)
- lng (DECIMAL(10,6)) — долгота
- coords_source (TEXT) — источник координат (address, area_{id}, none)
- address_raw (TEXT) — полный адрес (если указан)

**Работодатель**
- employer_id (TEXT)
- employer_name (TEXT)
- employer_accredited_it (BOOLEAN) — аккредитованная IT-компания
- employer_trusted (BOOLEAN) — надёжный работодатель

**Отрасль (только первая из списка)**
- employer_main_industry (TEXT) — название отрасли
- employer_main_industry_id (TEXT) — ID отрасли

**Профроль (только первая из списка)**
- professional_role (TEXT) — название роли
- professional_role_id (TEXT) — ID роли

**Требования и условия**
- experience_id (TEXT)
- experience_name (TEXT) — требуемый опыт
- employment_name (TEXT) — тип занятости
- schedule_name (TEXT) — график работы
- accept_temporary (BOOLEAN) — временная работа
- accept_labor_contract (BOOLEAN) — оформление по ТК
- internship (BOOLEAN) — стажировка
- night_shifts (BOOLEAN) — ночные смены

**Формат работы (объединены в строку через запятую)**
- work_format (TEXT) — удалёнка / гибрид / офис
- working_hours (TEXT) — рабочие часы в день
- work_schedule_by_days (TEXT) — график (5/2, 2/2)

**Навыки**
- key_skills (TEXT) — навыки через запятую

**Тесты и прочее**
- has_test (BOOLEAN)
- test_required (BOOLEAN)
- archived (BOOLEAN)
- response_letter_required (BOOLEAN)
- premium (BOOLEAN) — премиум-вакансия
- billing_type (TEXT) — тип размещения

**Пересчитанные поля (заполняются триггерами)**
- salary_from_rub (NUMERIC(12,2)) — зарплата "от" в рублях
- salary_to_rub (NUMERIC(12,2)) — зарплата "до" в рублях
- salary_from_net_rub (NUMERIC(12,2)) — чистая зарплата "от"
- salary_to_net_rub (NUMERIC(12,2)) — чистая зарплата "до"
- salary_avg_rub (NUMERIC(12,2)) — средняя зарплата в рублях
- salary_avg_net_rub (NUMERIC(12,2)) — средняя чистая зарплата

**Служебные**
- scraped_at (TIMESTAMPTZ) — дата загрузки в базу (по умолчанию NOW())

## Логика пересчёта зарплат (триггеры PostgreSQL)

### 1. Перевод в рубли

- Если валюта RUR — значения копируются без изменений
- Для других валют:
  - Маппинг: BYR (из HH) → BYN (в таблице курсов)
  - Поиск курса на дату публикации вакансии (ближайший предыдущий)
  - Если курс на дату отсутствует — берётся последний известный
  - salary_in_rub = salary_in_currency × rate_to_rub

### 2. Расчёт чистой зарплаты (прогрессивная шкала НДФЛ)

Шкала налогообложения с 2025 года:

| Годовой доход | Ставка |
|---------------|--------|
| до 2,4 млн руб. | 13% |
| 2,4–5 млн руб. | 15% на превышение |
| 5–20 млн руб. | 18% на превышение |
| 20–50 млн руб. | 20% на превышение |
| более 50 млн руб. | 22% на превышение |

Принцип расчёта:
- Годовой доход = месячная зарплата × 12
- Налог рассчитывается прогрессивно по диапазонам
- Месячный налог = годовой налог / 12
- Чистая зарплата = гросс зарплата − месячный налог

### 3. Средние значения

- salary_avg_rub = (salary_from_rub + salary_to_rub) / 2
- salary_avg_net_rub = (salary_from_net_rub + salary_to_net_rub) / 2
- Если одно из значений NULL — результат NULL

## Архитектура пайплайна

### Схема взаимодействия компонентов

**ИСТОЧНИКИ ДАННЫХ**

- HeadHunter API (вакансии, города, работодатели)
- ЦБ РФ (курсы валют)


**↓**


**GITHUB ACTIONS (оркестрация)**

| Скрипт | Назначение | Расписание |
|--------|------------|------------|
| daily_currency.py | Сбор курсов валют | Ежедневно, 07:00 MSK |
| daily_vacancies.py | Сбор новых вакансий | Ежедневно, 21:00 MSK |
| vacancies_historical.py | Сбор истории | Ручной запуск |


**↓**


**SUPABASE (PostgreSQL)**

**Таблицы:**
- exchange_rates — курсы валют
- vacancies — вакансии (49 полей)

**Триггеры (автоматическое обогащение):**
- Перевод зарплат в рубли по курсу на дату публикации
- Расчёт чистой зарплаты (НДФЛ по прогрессивной шкале)
- Расчёт средних значений (гросс и нетто)


**↓**


**YANDEX DATALENS (визуализация) — в разработке**

Дашборд с графиками:
- Динамика зарплат (гросс / net)
- Карта распределения вакансий по регионам
- Топ навыков и профессиональных ролей
- Аналитика по работодателям (отрасли, аккредитация)
- Временные ряды публикаций

## Процессы сбора данных

### Ежедневный сбор курсов валют (daily_currency.py)
- Запуск: 04:00 UTC (07:00 MSK)
- Источник: ЦБ РФ
- Действие: получение курсов за текущий день → upsert в exchange_rates

### Ежедневный сбор новых вакансий (daily_vacancies.py)
- Запуск: 18:00 UTC (21:00 MSK)
- Период: вчера и сегодня
- Логика:
  1. Сбор ID вакансий за период
  2. Фильтрация ID, отсутствующих в базе
  3. Загрузка полных данных по новым ID
  4. Пакетная вставка в vacancies (по 50 строк)
- Защита от дубликатов: проверка существующих ID перед вставкой

### Исторический сбор вакансий (vacancies_historical.py)
- Запуск: вручную из GitHub Actions
- Параметры: количество дней (по умолчанию 30)
- Логика: аналогична ежедневному сбору, но за указанный период

## Обогащение данных

| Объект | Источник | Логика обогащения |
|--------|----------|-------------------|
| Координаты | address.lat/lng или /areas/{id} | Если есть точный адрес — его координаты, иначе координаты города из справочника |
| Отрасль | /employers/{id} | Первая отрасль из массива industries |
| Профроль | professional_roles из вакансии | Первая роль из массива |
| Курс валюты | exchange_rates | Ближайший курс на дату публикации или последний известный |

**Кэширование:** координаты городов и отрасли работодателей кэшируются в памяти скрипта для уменьшения числа запросов к API.
